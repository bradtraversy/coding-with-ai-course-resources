# Planning a Project From Scratch

Now we start the real work of building our app. Unlike with "vibe coding", we don't want to just open up an AI prompt and say build this. We want to carefully think of and map out what we want.

Everyone has their own process for this and ultimately, yours may be different than mine. I think this is a good general workflow for planning any coding project whether you are using AI or not:

<img src="../images/planning.png" width="700" />

You don't have to follow this to a tee, but it's a good general guideline.

I usually start by creating a project spec file with all the main features that I want. Obviously, this can change, but you want to have a good idea of how your project should work and document it.

Typically you're going to write this yourself with as many ideas and features as you can think of. It's important to be specific when working with AI, otherwise, you're not the architect, the model is. The model should be the builder, you're the architect.

You will have an attached file to this lesson called `project-spec.md`. Download that and just put it in the `context` or root of your project for now. It doesn't really matter because we aren't keeping this version of the file. Like I said, normally you would type most of this out, but to save time, I included it for you.

This is an overall view of how our project should work. Look over it, but what I like to do is run it through AI to create a better version for us with even more depth to it. Then we can keep that in our persisted context as we build.
