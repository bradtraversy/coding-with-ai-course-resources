# Course Resources & Tools

I want to talk a little bit about the course resources and the tools you'll need.

## Course Resources

I'll have two Github repos for you guys. I'll also attach the files to this lesson for download.

### Course Files

First is the course files repo. This is not the project code, this is everything else including context files, custom commands and subagents that we create, feature specs, research documents, diagrams and everything that we use in the course. This has my entire AI workflow that is completely re-usable in other projects.

This will also include the **course-lessons**, which is basically a written version of the course. You can follow along with these markdown documents along with the videos.

### Project Files

Then the project repo is the actual DevStash code that was generated. It's important to understand that since most of the code is AI generated, when you follow along, your code could be different even if you used the same exact commands and the same AI tools. So just be aware of that.

## Tools You Need

You don't need much for this course in terms of tooling.

## Text Editor

Of course, you need a text editor or IDE. I assume most of you probably have an editor that you use. I'll be using VS Code.

## AI Tool

You'll need some kind of agentic AI tool, whether that's an all in one editor like Cursor, a CLI like Claude Code, which is what I'll be using, a text editor extension and so on. I'll be going over a lot of the current available AI tools in a little bit.

## Node.js

You'll also need Node.js installed. At least version 18, but I would suggest the latest supported version. You'll need this because we'll be using npm to install packages. We'll also be using Next.js as our framework.

## Git

You'll need Git installed on your machine. On MacOS, you can install it with Homebrew. You can also download from https://git-scm.com and install it.

## Terminal

You'll also need a terminal. You can use your standard Mac, Windows or Linux terminal. When I use Windows, I actually use a tool called Git Bash, that can be downloaded from the same https://git-scm.com website. I'll be using the standard zsh terminal within the VS Code integrated terminal.

That's pretty much it. This is very simple and standard.
