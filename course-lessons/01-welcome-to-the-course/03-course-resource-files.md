# Course Resource Files

Throughout this course, you'll hear me reference the course files or resource files regularly. Everything lives in the course repository, and I'll do my best to link the relevant files directly in each lesson.

Don't get overwhelmed by the volume of it upfront. A lot of this will get thrown at you quickly, but we work through it slowly and in context — it'll make much more sense once we're actually in the workflow.

Here's a breakdown of what's in the repo and what each piece is for.

## `/course-lessons`

A written version of the entire course from start to finish. Use these alongside the videos as a reference or supplement — what I'd normally call premium docs in my other courses.

## `/diagrams-notes`

All the Excalidraw diagrams I use throughout the course, exported as PNG images. If you want them for your own notes or reference, they're here.

## `/context`

This is one of the most important folders. In simple terms, context is what the AI holds in memory. The files directly in this folder get loaded automatically when Claude Code starts up — things like how I want the model to interact with me, our coding standards, and a project overview.

The most important file here is `current-feature.md`. We'll be updating this constantly once we get into our workflow — before and after nearly everything we do.

### `/context/features`

Our workflow for adding features centers on feature spec files. For simple tasks and early prototyping we'll use plain prompts, but most of the time we'll be loading from these spec files into the current feature and then implementing from there. We have a custom `/feature` skill to load these files and handle the different stages — start, test, review, and so on.

### `/context/fixes`

Fixes that get added to the current feature file for implementation.

### `/context/research`

Research files used for generating documentation. We have generated docs for major parts of the project — auth integration, CRUD operations, Stripe, AI features, and more. The `/research` command loads these.

### `/context/screenshots`

UI screenshots used as visual reference during development.

## `/skills`

Custom commands for our workflow. In Claude Code these are called skills. The ones we use most:

- `/feature` — the backbone of our workflow; used to add, implement, and test features
- `/research` — generates documentation for a given area of the project
- `/cleanup` — handles housekeeping tasks like ordering the changelog and finding unused files
- `/list-component` — a simpler one I built to introduce the concept of skills

There's also a `legacy-custom-commands` folder showing how slash commands worked before they were folded into the skills system.

## `/custom-subagents`

Subagents are spawned with their own context window to handle specific tasks independently. We use them primarily for code and UI auditing. Other tools may call these something different, but the concept is the same across the board.

## `/prompts`

The prompts we use directly — mostly for cases where we're not working from a feature spec file. Some of these are long, so rather than typing them out, you can copy them from here.

## `/docs`

This is where documentation generated by the `/research` command lives. These files are written for humans, not fed to the AI.

## `CLAUDE.md`

The main context file for Claude Code. Whatever you put in here gets loaded into the model's context window on startup. Think of it as your standing instructions for every session.
